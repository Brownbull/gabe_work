## üÜò Common Questions

**Q: Do I really need to tailor every CV?**
A: YES! Tailored CVs have 3-5x higher response rates. 30-60 min investment is worth it.

**Q: What if I don't have a required skill?**
A: Options:
1. Don't apply (if it's truly critical and you have zero related experience)
2. Address proactively: "Strong Python and C experience; currently learning Kubernetes"
3. Emphasize related skills: No Terraform? Emphasize infrastructure-as-code mindset

**Q: How long should my CV be?**
A: 2 pages for 10+ years experience (you). 1 page is too short and undersells you.

**Q: Should I include mainframe experience?**
A:
- YES for: Mainframe roles, legacy modernization, fintech
- NO for: Startups, modern SaaS, frontend-heavy roles
- Use space for more relevant modern tech

**Q: Do I need a cover letter?**
A: Depends:
- YES if: Small company, hiring manager reviews, easy to find contact
- NO if: Large company, ATS-heavy, generic "Apply" button
- Make it short (3 paragraphs) if writing one

**Q: What about gaps in employment (post-2025)?**
A:
- Be honest: "Career transition focusing on skill development"
- Show activity: Personal projects, learning, certifications
- Don't apologize: Frame positively
